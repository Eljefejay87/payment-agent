## Persistent `/data` State

Railway must mount a persistent volume at `/data`.

Voicemail runtime state:

```dotenv
VOICEMAIL_DATA_DIR=/data
VOICEMAIL_STATUS_PATH=/data/voicemail_status.json
VOICEMAIL_RUNTIME_STATE_PATH=/data/voicemail_runtime_state.json
VOICEMAIL_HEALTH_PATH=/data/voicemail_health.json
```

`voicemail_runtime_state.json` stores:

- `processed_voicemail_ids`
- `last_successful_scan`
- `pending_callbacks`
- `callback_completion_status`
- `last_teams_summary_sent_for`

`voicemail_health.json` stores:

- service status
- last successful scan
- last successful job
- records processed by the last successful job
- last error details
- process ID

## Duplicate Protection

The agent uses Outlook `internetMessageId` first and Graph `id` second. During a scan it skips:

- IDs already present in persisted runtime state.
- Duplicate IDs appearing more than once in the same Graph result set.

Runtime state is written only after parsing succeeds. If Railway restarts after a successful scan, the next scan reads `/data/voicemail_runtime_state.json` and skips previously processed voicemail IDs.

Voicemail Phase 1 has no Teams alert sender. No duplicate Teams alerts can be generated by the current implementation. The runtime state includes callback and summary guard fields so future Teams behavior can use the same durable idempotency layer.
